# Divulgação responsável (security)

## Como reportar

1. **Preferido:** no GitHub do repositório, active **Private vulnerability reporting** (*Settings → General → Security → Vulnerability reporting*) e use **Security → Report a vulnerability**. Não publique pormenores de exploração em *issues* abertos.
2. **Alternativa:** contacto privado acordado pela equipa (e-mail ou canal interno).

## Âmbito

Aplicação web (Vite/React), API Node opcional (`server/`), Supabase (Postgres com RLS, Storage, Edge Functions), integrações (Mercado Pago, webhooks, terceiros referidos no código).

## Expectativas

Confirmamos receção quando possível. Pedimos coordenação antes de divulgação pública até existir mitigação ou prazo acordado.
